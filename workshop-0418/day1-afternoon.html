— 这一下午的主线一句话

这一下午在讲什么

上午讲的是"认知反转"——一号位必须先在脑子里完成翻转。下午一坐下来，Vivian 很明确地切到了下一档：从认知到手上。

她的逻辑是，认知再清楚，如果手没有练过一遍，所有的话都会停留在感慨层面。所以整个下午几乎都在做一件事——让每个人现场把至少一个真实业务场景跑通成一个 skill。

午饭前布置的"Apple 风格个人 HTML"作业先验收（大部分人都跑出来了、有人甚至跟 Vivian 生成的是同一个模板），然后四件套再讲一遍，接着就是各种真实场景的现场 demo：简历筛选、面试录音、4 万字讲稿整理、英文文章深度学习笔记。

01 工具再过一遍

四件套：为什么是这四件

午饭前很多人刚装完工具，下午 Vivian 花了一段时间把四件套再过一遍，因为后面所有动作都在这四件套的基础上展开。

工具 角色 Vivian 的原话
Claude Code 大脑 "这一整套的核心"
Obsidian 文件阅读器 + 世界模型的家 "他打开的是我们本地的一个文件夹，你希望他阅读什么，就给他文件夹里放什么东西"
Claudian（插件） 让 Claude Code 在 Obsidian 里更好看 "没有这个插件一样能用，对用惯命令行的人来说没差别"
闪电说（语音输入） 无延时把想法变文字 "因为如果没有输入，对 AI 来说就是什么都没有，一定要留下痕迹"

语音、文字、图片、文档，任何形式的痕迹都要留下，因为 AI 认识你的全部依据就是你留下的痕迹。

模型选择的几个判断

Claude Opus 4.6 / 4.7 是目前最强的模型（Vivian 自己和身边朋友用下来的判断）。Codex（GPT）在写代码上也很强，两者可以互补——但如果 Claude 能用就优先用 Claude。

关于国内模型和 Claude 的差距，她讲得很直白：

"从豆包到 Claude，Claude 就是变了一个人——把它当成一个拟人化的数字物种的话，它就是变了一个人。"

她引用了阿里千问负责人自己的说法——差距可能有 1-2 年。所以只要有渠道就用；100 美金不够就上 200 美金。

迁移的安全感

大家今天装好的这些东西，以后都可以迁移到其他 harness（如爱马仕、小龙虾）——只要对 Claude 说"我要搬到 X 去了，你帮我把这些文件变成 X 能读的格式"，它就会帮你搞定。

在一个土壤上养好你所有的东西，不要焦虑。

02 第四个文件

三剑客之外——SKILL 是什么

上午讲过三剑客（USER / MEMORY / CLAUDE），下午 Vivian 重点展开了第四个文件：SKILL.md。

打个比方：skill = 岗位 SOP

如果说 USER.md 是身份档案、CLAUDE.md 是入职手册，那 skill 就是每个岗位的 SOP。我们公司里每一个岗位都有一堆 SOP——市场部的文案怎么写、HR 怎么面试、财务怎么报销——AI 干活的时候也需要这样一份 SOP，告诉它"这件事你过去怎么做的、应该按什么流程做"。

Skill 的三个来源

来源 做法 场景
直接让 AI 找 "帮我找一个做 PPT 的 skill" 通用场景
GitHub 开源 Anthropic 官方有 "ocean agent skill"，非常多好用的 skill，给它链接让它只在这里找 有明确需求但不想从零写
自己写 自己的业务流程，通用 skill 不够用 个性化业务，如尹亚楠的 AI 答疑 skill

尹亚楠的案例特别说明了"为什么必须自己写"——她用了大量自己积累的知识库、自己的书、自己十年答疑的语料，这些东西 GitHub 上找不到，只能自己写，或者交给专业人员定制。

⭐ 什么值得做成 skill — 第二次法则

需要做第二次的事情，都值得做成一个 skill；需要用电脑完成的事，都可以做成一个 skill。

但 skill 有优先级。按以下四条同时满足的程度排序：

1
高频
高频发生
2
重复
高度重复
3
可判断
有判断标准（做得好和不好有客观差异）
4
高价值
能带来高价值

满足得越多，越值得把它封装成可复用的 skill。

🚨 但同时——不要装 100 个 skill

现场有同学在后半段专门讲了这点：装 100 个 skill 不一定是好事。因为 AI 每次开场要扫一遍所有 skill 的"简介"，skill 越多它脑容量的负担越重。只装你真的会用的。

03 实战第一轮

每个人现场做一个自己的 skill

Vivian 发了一个"母提示词"到微信群，让每个人把自己最耗时、最重要、但过去只能人工做的一个场景扔进去，当场封装一个 skill。10 分钟后几乎每个人都跑出了东西。

王龙的"金一南讲稿整理 skill"

王龙是做企业家培训的，每次请完大牌讲者（上周末请的金一南将军），讲者会拍着他肩膀说"回头给我整理一个讲稿"。一个 3 小时的讲稿 ≈ 4 万字，过去他的工作流是：

1
几年前
外包团队分段听录音、人工誊写，好几个人做，靠人工校对
2
讯飞时代
讯飞转写 → 扔豆包或 DeepSeek 校正 → 但 4 万字会崩、会出错，Kimi 超过几千字就崩
3
今天现场
4 万字扔给 Claude → 细致整理 → 第一版有乱码 → 王龙说"乱码太多你想办法" → Claude 自己分析乱码原因、全部处理干净 → 当场说"以后还要重复做，给我封装个 skill" → Claude"连个磕巴都没打"生成了 transcript organizer skill

"你说它值钱不值钱？它一点钱都不值。但你说它重要不重要？它很重要。因为他未来我要想再请的话，这个文档给到他，会说你这个整得太好了——我就说我花了两个星期。这就很重要。"

这是一号位 skill 的本质：小而重要的事——对外体现你的专业度和用心，对内不值得你花两天，过去只能硬扛，现在一次封装永久复用。

第二个案例：飞书云文档进知识库

现场 demo

有一位同学（用了录音豆+飞书的）现场做了一个 skill：从飞书自动爬他的云文档变成 MD，下载到本地知识库。原理是飞书开放了 API / CLI 接口，所以 Claude 可以直接调接口去取。

Vivian 补了一个价值判断：飞书豆的录音是可以的，但它自动生成的会议纪要不要信任——它走的是豆包模型，总结质量达不到要求。

正确用法：用飞书豆/通义录音 → 只取原文 → 喂给 Claude 生成纪要。再往下一步：如果你带电脑开会，根本不需要飞书豆——Vivian 新做的实时录音 skill 在电脑本地直接跑（Whisper 模型），录音 + 转写 + 生成纪要一条龙。

04 招聘流水线 demo

Vivian 的招聘 skill — 全下午最精华

这是 Vivian 为自己正在招的 AI Agent 产品经理岗搭的一套完整招聘流水线。讲完你会非常直观地感受到——AI 不是帮你效率 +30%，是把整个工作形态换了个物种。

背景：新岗位的人才市场在变

Vivian 在 Boss 上发了一个 AI Agent 产品经理的 JD，收到 100 份简历，质量都蛮高。她对比了一下：过去招过 AI 人才时简历寥寥无几，现在市场在变——AI 人才变多了，这个岗位没有想象中难招。

完整流水线（每一步都封装成 skill）

1
第一步 · 简历筛选
把 7 份候选简历存本地，把 JD 也喂给 AI → 根据 JD 匹配度自动分梯队：第一梯队（必约）、第二梯队（可约）、第三梯队（不用推荐）。过去肉眼看 100 份要半天，现在 10 分钟出报告。
2
第二步 · 出面试题
AI 看过所有简历之后，针对具体候选人出一份专属面试题：开场白怎么说、项目怎么深挖、核心考察点。甚至主动提醒："他的意向地点是杭州和深圳，你在上海，一定要问这个问题"。
3
第三步 · 面试实时录音
面试开始前启动录音 skill。过程中候选人讲到专业术语没听懂 → 现场对 AI 说"刚刚那个术语我没听明白，你帮我查一下" → AI 实时给你生成文档链接。这是飞书录音做不到的——飞书没有实时。面试完当场 AI 打分 + 填面试备忘 + 生成实操测试题。
4
第四步 · 记录与题库沉淀
所有人的筛选情况、面试情况都进记录文档。面完 10 个人之后，AI 自动形成公司自己的面试题库。这个题库可以复用给其他面试官——你可以把顶级 HR 的面试过程录音、封装成 skill，让普通 HR 按这个方法面。

为什么要把顶级员工的工作封装成 skill

"顶级员工很难复制，但 skill 可以批量发。我把顶级 HR 面试的过程、筛选标准封装好，就可以批量给 A、B、C 员工使用。"

这就是 Vivian 反复说的 "Skill = 数据员工" 的真实含义——公司最优秀的那个人的工作方式，用 skill 复制出无数个他。

可能的下一步：打通 Boss 直聘

现在 Vivian 的简历还是 HR 手动下载再喂给 Claude 的。理论上可以通过 MCP 或浏览器操作直接自动拉简历——"让 Claude 自己去找有没有开源的 Boss 直聘接口"，如果有就直接用，没有就让 AI 通过模拟用户界面操作去做。这一步留给未来迭代。

05 信息 → 沉淀

Web Clipper + 视频：把消费变沉淀

过去：刷到好文章 → 点赞收藏 → 再也不看。现在：一键 Add to Obsidian → AI 自动翻译/提炼/分类 → 深度笔记 → 卡片分享到俱乐部。

Vivian 现场演示流程

  1. 打开微软 AI 战略的英文文章
  2. 点一下 Web Clipper 插件（Chrome/Edge 都有）→ 自动到 Obsidian
  3. 对 Claude 说："帮我翻译，做中英文对照，深度学习笔记，我想分享给俱乐部"
  4. Claude 同时做了三件事：
    • 中英文对照原文（还会自动标红重点）
    • 深度笔记：吸收点、文章结构、主段分析、架构分析
    • 跟 Vivian 业务结合的提炼——这家公司（Block）的做法对"一号位 AI 进化论"项目有哪些启示
  5. 再对 Claude 说："帮我做成卡片"→ 直接做成可转发到微信的卡片

为什么这件事重要

因为 AI 认识你。同样一篇文章，喂给 ChatGPT 只是摘要；喂给认识 Vivian 的 Claude，它会自动做这些事：翻译成中英文对照、提炼出和 Vivian 自己"Company World Model"工作相匹配的点、同时生成三个版本（俱乐部分享稿、公众号素材、个人学习笔记）——所有这些都不用再招翻译、提炼人、设计师。

"为什么我一个人可以——这个团队主要就我一个人做——相当于他真的是放大了我十倍的功效。只要我有想法，他就可以帮我执行完成。"

视频场景的对应工具

工具 场景
Get 笔记 B 站视频：给链接 1-2 分钟出全文
抖音 skill（Vivian 自己封装了） 抖音链接 → 全文 → 梗概
NotebookLM 把优秀视频喂给它学习，可以在 YouTube 里对内容提问
Claude 原生 小宇宙播客 2 小时听不完 → 复制到 Claude 直接解读

核心认知：视频/音频/文章都可以变成 AI 的养料，关键是"喂进去"这一步。

06 新业务样本

蔡一鸣的 AI 博主事业部

中间穿插的一段，蔡一鸣分享了他正在跑的 AI 博主事业部——这是现场对"AI 如何创造新业务线"最具体的一个案例。

他看到的两个信号

1
需求端在爆炸
AI 硬件/软件测评视频的需求一个月能接四五单都接不完，每周都有
2
供给端可以无限降成本
他的博主用 AI 产初稿、配图、剪辑，一个人可以带 5 个 agent 并行跑

他的逻辑：需求端确定的快速通告类测评（几万块、3 万块以内的订单），客户不会要求非常高的质量——这种订单完全可以让 AI 写初稿，人工只做终审和少量剪辑。

"有了 AI 加速产能的提升，我找到了确定的需求，产能端我找了一个更低成本的方式。一个人带 5 个 agent 就可以干。"

所以他把这件事做成了一个事业部，正在落地。

Vivian 的点评

她把这个案例和同学"晴空"的做法归到同一条线：底层都是用 AI 把一个过去劳动密集的行业变成低成本高杠杆的方式。差别只是前台的品类——晴空是带货，蔡一鸣是内容营销，LIPRO 是硬件开发，方法论是同一套。

这就是"反转 B（管理幅度 ×10）"在真实业务上的具体呈现。

07 Karpathy 方法论

从 AI 原理层证明"必须自己下场"

这一段是下午的理论骨架——为什么我们今天这么辛苦地装工具、写 skill。

核心认知① AI 其实没有任何记忆

你每次和他的对话，他都是从零开始给你回复。你有一种幻觉——他非常了解你、了解你之前跟他干了什么——是因为你非常贴心地帮你把相关的信息给他看了。

大模型本质是一个概率机器：你给它一个输入，它按概率输出一段文本。你体感上它是"懂你的"——那是因为背后有个系统把记忆补给了它。

核心认知② 模型决定天花板，Harness 决定落地率

M
大模型
一个脱缰的野马——能力强但无记忆、短上下文、容易跑偏
H
Harness
驯服这匹野马的系统——每次对话都帮你补齐它需要的所有上下文

闭源最强两家：Codex（OpenAI）+ Claude Code（Anthropic）——都是 harness + 模型一整套。
开源代表：小龙虾、爱马仕——harness 开源给你，模型你可以自己选。

核心认知③ 上下文就是 AI 的脑容量——要善待它

每次和大模型对话，它的"脑容量"（上下文窗口）——听起来很长，对一个超大项目来说非常短。所以必须做渐进式披露（Progressive Disclosure）：

  • Skill 的设计原则：AI 第一眼看到的不是 skill 内部所有内容，而是一个简介标签
  • 只有当语义匹配了（比如你说"做一个 PDF"），它才会打开 PDF skill 的详细内容
  • 这就是为什么装 100 个 skill 不是好事——每个 skill 的简介都要占脑容量

类比：员工的工作台上可以有 100 个工具，但每次上班第一眼是看工具的标签，而不是把每个工具的说明书都读一遍。

核心认知④ 不能全交给 AI——一号位必须保持掌控

当你决定把一件事情完全给 AI 的时候，AI 在做的时候，有无数个微小选择都是按它的品味去选的。你对产出物的掌控率会非常低。AI 做出的能力不是你的，你也非常没有掌控感。你就是一个不懂食物的厨师。

所以要限制自己的野心：

  • AI 可以帮你，但你要通过亲自做任务的切身体验才能生发出新思考
  • 新的跨任务理解、新的创新点，必须是你自己跑过一遍才能长出来的
  • 完全交给 AI = 你变成一个不懂食物的厨师，下不了判断

这一段和 Vivian 上午讲的 "反转 C：创始人必须自己下场" 是一个意思——只是这次是从 AI 的原理层面证明了为什么必须自己下场。

Karpathy 的 vault 方法论

Vault 骨架 · 三条规则

Karpathy（前 OpenAI 合伙人、Tesla AI 负责人）在 GitHub 发了一套 vault 搭建的 markdown 文档。现场这位同学把它改造后用在了自己的笔记系统里：

raw/ 文件夹：永久保存的原始资料，不能改。

wiki/ 文件夹：从 raw 派生出来的结构化内容，可以改。

固定规则：回答问题时，AI 先看 wiki，wiki 不够再去看 raw，按约定的顺序翻看文件夹。

💡 Karpathy、Vivian 的模板都可以参考，但最终需要整理自己的。

08 执行纪律

红黄绿灯系统

讲到自己的 CLAUDE.md 时，Vivian 专门展开了红黄绿灯系统——这是她用了很长时间才沉淀下来的执行纪律，强烈推荐同学们建自己的版本。

🟢 绿灯（AI 自己跑，不问）

  • 读文件、搜索信息、整理资料
  • 跑流水线（公众号、会议处理、客户调研）
  • 创建新文件、保存到 output/
  • 生成封面、截图、HTML 转换
  • 公众号写到草稿箱（这一条是 Vivian 特意开的绿灯，发布到后台不用等确认）

🟡 黄灯（做完提醒，不打断）

  • 涉及客户/嘉宾名字出现在公开内容里 → 做完提醒"要不要隐去"
  • 大幅修改现有文件 → 自动另存新版本
  • 不确定的业务判断（如文件放哪个夹）→ 先问一下

🔴 红灯（必须问过才能做）

  • 涉及钱：付款、定价、合同条款
  • 对外发布：发飞书、发朋友圈、发邮件
  • 不可逆删除：原始文件不能删
  • 底层架构变更：改 CLAUDE.md 核心规则、改文件夹结构
  • 涉及第三方：代你联系客户、回复别人

为什么这套系统重要

AI 的能力是把双刃剑——它能帮你批量做事，也能批量犯错。红黄绿灯系统本质上是把你对每类任务的风险判断预先固化下来，让 AI 在没有你的情况下也能按你的价值观做决策。

这套系统建好之后，90% 的日常任务可以一口气跑完，不再需要你一步步确认。

09 防分叉机制

backlog：大任务总会长出小分叉

Vivian 在讲 MEMORY.md 时专门介绍了一个她最近发现的好用功能——backlog。

场景

她在准备这次工作坊课件时遇到的真实问题：

  • 正在准备课件 → 冒出一个公众号选题
  • 准备课件 → 想到一个 PPT 的优化方向
  • 准备课件 → 看到一篇文章想深入探讨

一个主线任务，会同时分叉出 12 个、3 个、4 个子任务。如果全部当场处理，主线永远完不成；如果不处理，这些想法就丢了。

解法：backlog.md

对 Claude 说："把这个记到 backlog"——它会生成一个 backlog.md 文档，记录：

  • 日期 + 任务名
  • 来源话题（从哪个主线冒出来的）
  • 为什么值得做
  • 触发时机（什么时候回来做）

下次问起来时，不用再给它复述背景——它从 backlog 里直接读。

适用场景：一个大任务里有 10+ 子任务要做的时候，让 Claude 生成类似交接文档的 backlog 特别好用。

6 条 Takeaway

1

这周五前跑 5 个 skill

不求完美，就按"第二次法则"——每次做一件事做第二遍的时候，停下来封装成 skill。一周下来会有 5 个。复用 3 次以上的就继续迭代，用不上的删掉。

2

复刻 Vivian 的招聘 skill 全链路

如果你在招人（几乎每个创始人都在招），把这一套拉出来直接用：简历存本地 + JD 喂 AI → 分梯队；针对每个人定制面试题；面试实时录音 skill；结束自动打分 + 生成实操测试题；10 个人之后沉淀公司面试题库。

三个月后，招聘质量一定会上一个台阶，而且可以复用给其他面试官。

3

建红黄绿灯系统

把 Vivian 那套直接拿过来改——写进你的 CLAUDE.md：绿灯日常内容生产、文件处理、跑流水线；黄灯涉客户/嘉宾信息、大幅改文件；红灯钱、对外发布、删除、架构变更。

不建的话，AI 要么每步都问让你崩溃，要么擅自做了让你后怕。

4

装 Web Clipper，把信息消费变成沉淀

Chrome/Edge 装 Web Clipper → 看到好文章一键到 Obsidian → 对 Claude 说"帮我做中英文对照和深度笔记"。一个月之后你的 vault 里会多出几十篇深度笔记，而不是几百篇未读收藏。

5

不要装 100 个 skill

Skill 不是越多越好，每个 skill 都占 AI 的脑容量。只装你真的会用的，用不上的定期清理。

6

最重要的一条：别完全交给 AI

这一下午最底层的道理只有一条：AI 放大你，但不替代你下判断。你必须亲自跑一遍才能长出新的跨任务认知。完全交给 AI = 不懂食物的厨师。
